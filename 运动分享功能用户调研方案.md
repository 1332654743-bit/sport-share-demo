# 运动分享功能用户调研方案

## 一、调研目标

通过行为观测+极简主观题，验证三个核心问题：

1. **不同运动场景下，用户更倾向哪类分享样式？**
2. **用户是否愿意编辑，以及愿意花多久编辑？**
3. **如果编辑，用户真实优先使用哪些功能？**

---

## 二、调研方法

**基于现有 Demo 的行为埋点测试 + 场景结束后极短主观题**

核心原则：让用户"做"而非"说"——通过操作行为数据得出结论，主观题仅用于交叉验证。

---

## 三、调研流程

```
功能说明页（30秒，了解三种分享方式的形态）
    ↓
场景任务（3个场景，顺序随机化）
    ↓ 每个场景内部流程：
    ├── 阅读场景描述（具象化情绪/环境）
    ├── 选择分享样式
    ├── 进入 Demo 原有流程，自由操作
    │   （顶部始终有「分享」和「返回」按钮，用户随时可结束）
    └── 1道场景结束主观题
    ↓
结尾：2道开放题
```

**预计总时长：8-12分钟/人**

---

## 四、功能说明页设计

**目的**：让用户知道有哪三种分享方式，但不教具体操作。

**内容**：一屏展示三种分享方式的效果预览图+一句话说明

| 分享方式 | 一句话说明 |
|---------|-----------|
| 水印海报 | 一张精美图片，贴上你的运动数据 |
| 轨迹视频 | 你的运动轨迹变成一段动画视频 |
| 运动报告 | 完整的运动数据长图报告 |

**注意**：不做功能引导，不点击进入体验。用户只需要知道"有这三种东西"即可。

---

## 五、场景设计

### 场景1：日常打卡

> **场景描述：**
> 
> 周三晚上，你刚在小区旁边跑完5公里，出了一身汗。你掏出手机走在回家路上，想发个朋友圈记录一下今天的运动，然后到家赶紧洗澡。

**模拟数据**：5.02km / 32分钟 / 配速6'23" / 心率155bpm

---

### 场景2：成绩突破（PB）

> **场景描述：**
> 
> 今天你跑出了个人5公里最好成绩！比上次快了整整40秒。你停下来喘着气，心情非常激动，想立刻分享给跑友群和朋友圈，让大家看看你的进步。

**模拟数据**：5.00km / 24分18秒 / 配速4'52"（PB标识） / 心率172bpm

---

### 场景3：马拉松完赛

> **场景描述：**
> 
> 你完成了人生第一场全程马拉松！领完奖牌后坐在完赛休息区，喝着水，腿酸但内心充满成就感。这是一个里程碑时刻，你想认真做一条内容，发朋友圈/小红书/视频号纪念。

**模拟数据**：42.20km / 4小时38分 / 配速6'35" / 心率148bpm / 累计爬升186m

---

### 场景顺序随机化

每位用户看到的三个场景顺序随机排列（共6种排列方式），消除学习效应。数据分析时可按"该场景出现在第几位"做分组验证。

---

## 六、场景内部流程及数据采集

### 第一步：选择分享样式

用户阅读场景后，看到三个分享样式入口（水印海报 / 轨迹视频 / 运动报告），自行选择。

**采集数据：**
- 用户选择了哪种样式
- 从看到选项到做出选择的决策时长

---

### 第二步：自由操作（还原 Demo 原有流程）

选完样式后，用户直接进入 Demo 的原有编辑/预览流程。**不做任何人为干预**——Demo 本身顶部操作栏已有「分享」和「返回」按钮，用户随时可以分享结束，也可以继续编辑。

**核心原则：不问"你要不要编辑"，让行为自己回答。**

**行为推断逻辑：**

| 用户行为 | 推断 |
|---------|------|
| 进入后 <5秒就点分享 | 不需要编辑，默认效果满足需求 |
| 修改0个默认值就点分享 | 默认效果足够，该功能可简化 |
| 修改了N项再分享 | 需要编辑，按操作顺序推断优先级 |
| 停留很久但没改东西 | 可能在犹豫/找功能（需结合录屏分析） |
| 点返回放弃本次分享 | 流程受阻或效果不满意 |

**页面分类：**

| 页面 | 分类 | 理由 |
|------|------|------|
| PAGE 3C 预览海报 | 编辑预览页 | 有分享按钮，边编辑边预览，含多个编辑功能入口 |
| PAGE 4 轨迹视频编辑 | 编辑预览页 | 顶部有分享按钮，边编辑边预览视频 |
| PAGE 5 运动报告编辑 | 编辑预览页 | 顶部有分享按钮，边编辑边预览报告 |
| PAGE 3 选择模板 | 过程页 | 无分享按钮，必须选完才能进下一步 |
| PAGE 3B 选择背景 | 过程页 | 无分享按钮，是模板选择后的中间步骤 |
| PAGE 4D 添加图片 | 过程页 | 无分享按钮，是编辑页内的子功能页 |

> 分享弹窗（3D/4C/5B）和确认弹窗（4A/5A）属于系统交互层，不纳入采集分类。

**各类页面采集策略：**

#### 编辑预览页（PAGE 3C / PAGE 4 / PAGE 5）

| 采集维度 | 说明 |
|---------|------|
| 使用了哪些编辑功能 | 是否触发了有效操作变更（非误触） |
| 每个功能的操作时长 | 从进入到完成该功能的时间 |
| 功能使用顺序 | 用户的自然操作路径 |
| 从进入到点击「分享」的总时长 | 编辑时长 |
| 是否点击「返回」放弃 | 放弃节点 = 流程问题点 |

#### 过程页（PAGE 3 / PAGE 3B / PAGE 4D）

| 采集维度 | 说明 |
|---------|------|
| 停留时长 | <3秒 = 可能认为此步多余 |
| 是否修改默认值 | 直接下一步 = 此页可折叠为可选项 |
| 修改决策时长 | 花多久做出选择 |
| 是否从后续页面回退 | 回退 = 流程顺序可能需调整 |

---

### 第三步：场景结束主观题（1道）

> "刚才的操作中，哪个环节对你帮助最大？"（单选）
> 
> - A. 选择分享样式（海报/视频/报告）
> - B. 选择模板/风格
> - C. 调整展示数据/图表
> - D. 换背景/调颜色等视觉编辑
> - E. 不需要编辑，默认效果就好

**目的**：与行为数据交叉验证。行为高+主观高=确认高优；行为高+主观低=可能是路径强制经过。

## 八、数据分析框架

### 埋点标记策略

每条行为数据统一携带 `scenario` 字段（daily / pb / marathon），分析时支持两种视角：

| 视角 | 用途 | 方法 |
|------|------|------|
| **分场景对比** | 验证不同场景下行为是否有显著差异 | 按 scenario 分组统计 |
| **加权合并** | 输出整体功能优先级排序 | 按真实场景频率加权（如日常60% / PB25% / 马拉松15%） |

> 加权比例来源：调研前通过筛选问卷获取用户近1个月三类场景的实际分享频次，作为权重依据。

---

### 核心问题1：不同场景下用户更倾向哪类分享样式

| 分析维度 | 数据来源 |
|---------|---------|
| 各场景下三种样式的选择比例 | 第一步选择数据 |
| 选择决策时长 | 犹豫时间长 = 需求不明确 |

**预期输出**：场景×样式的选择矩阵（如"日常打卡：报告72% / 海报20% / 视频8%"）

---

### 核心问题2：用户是否愿意编辑 + 愿意花多久

| 分析维度 | 数据来源 |
|---------|---------|
| 各场景从进入到点击分享的总时长 | 第二步时长记录 |
| 各场景中修改默认值的项数 | 0项=不编辑，≥1项=编辑 |
| 各过程页是否使用默认值 | 过程页默认值修改率 |
| 各场景放弃率（点返回退出） | 第二步放弃数据 |

**预期输出**：
- 各场景编辑深度分布（如"日常：70%改0项 / 20%改1-2项 / 10%改3+项"）
- 各场景中位操作时长（从进入到分享）
- 过程页必要性评估（默认值修改率<20%的页面建议折叠/省略）

---

### 核心问题3：用户真实优先使用哪些功能

| 分析维度 | 数据来源 |
|---------|---------|
| 各功能使用率 | 有多少比例用户触发了该功能 |
| 各功能操作时长 | 花多久 = 投入度 |
| 功能使用顺序 | 先用的 = 优先级高 |
| 主观验证 | 场景结束后主观题 |

**预期输出**：功能优先级排序表

| 功能 | 使用率 | 平均操作时长 | 主观认可 | 优先级 |
|------|:------:|:----------:|:-------:|:-----:|
| 选择模板 | 85% | 8s | 高 | P0 |
| 修改数据项 | 62% | 12s | 高 | P0 |
| 换背景 | 45% | 6s | 中 | P1 |
| 颜色编辑 | 12% | 15s | 低 | P2/可砍 |
| ... | ... | ... | ... | ... |

## 九、样本量建议

| 方案 | 样本量 | 说明 |
|------|:------:|------|
| 主力调研（Demo行为测试） | 15-20人 | 场景顺序随机化，需覆盖不同运动频率的用户 |
| 补充验证（真实运动后操作） | 3-5人 | 种子用户运动结束后5分钟内操作，验证编辑时长系数 |

**用户筛选条件**：
- 每周至少运动2次
- 有过运动分享行为（近1个月内至少分享过1次）
- 覆盖不同运动类型（跑步/骑行/健身）

## 十、调研执行 Checklist

- [ ] 在 Demo 中植入行为埋点（点击事件、时间戳、页面切换）
- [ ] 实现场景顺序随机化逻辑
- [ ] 确认 Demo 各页面顶部「分享」「返回」按钮均可正常点击并触发埋点
- [ ] 过程页记录默认值是否被修改
- [ ] 设计3个场景的模拟运动数据
- [ ] 准备场景描述卡片（调研时展示给用户）
- [ ] 招募测试用户（15-20人）
- [ ] 准备数据记录表格模板
- [ ] 补充验证：联系3-5名种子用户安排真实场景测试

## 十一、预期交付物

1. **行为数据报告**：场景×样式选择矩阵、编辑意愿比例、功能使用率排序
2. **功能优先级建议**：P0（必须保留）/ P1（可保留可简化）/ P2（建议砍掉或隐藏）
3. **流程优化建议**：哪些过程页可以省略/合并，基于默认值修改率数据
